# 🗺️ رحلة الطالب الكاملة + صلاحيات كل دور في إتقان

---

## 🎓 أولاً: رحلة الطالب من أول ما يسجل

### المرحلة 1️⃣ — التسجيل والتحقق

```
الطالب يفتح الموقع → يضغط "تسجيل جديد" (/register)
       ↓
يدخل: الاسم + الإيميل + كلمة المرور + الجنس
       ↓
📧 يوصله إيميل كود OTP (صالح 24 ساعة)
       ↓
يدخل الكود في صفحة /verify
       ↓
✅ الحساب مفعّل — دور: student
```

---

### المرحلة 2️⃣ — رفع التسميع

```
الطالب يدخل لوحته → يضغط "رفع تسميع" (/student/submit)
       ↓
صفحة RecitationRecorder:
  - يختار: السورة + من آية → لآية
  - يختار نوع التسميع: تلاوة / حفظ / مراجعة
  - يسجّل الصوت من الميكروفون مباشرة
       أو
  - يرفع ملف صوتي جاهز
  - يكتب ملاحظات اختيارية
       ↓
الملف يُرفع على UploadThing
       ↓
📝 يُسجّل في DB جدول recitations بحالة: PENDING
       ↓
🔔 إشعار يذهب للأدمن: "تسميع جديد غير معين"
```

---

### المرحلة 3️⃣ — انتظار التعيين (Admin يتدخل)

```
التسميع في حالة: PENDING
       ↓
الأدمن يدخل على /admin/recitations
يشوف التسميع الجديد بدون مقرئ
       ↓
الأدمن يختار مقرئ مناسب من القائمة
ويعيّنه على التسميع
       ↓
📝 حالة التسميع: PENDING (ما زالت)
🔔 الأدمن يُشعر المقرئ: "تسميع جديد معين عليك"
```

> **ملاحظة:** فيه آلية `is_accepting_recitations` — لو المقرئ غلّق قبوله، الأدمن ما يعيّنه تسميعات جديدة.

---

### المرحلة 4️⃣ — المقرئ يراجع التسميع

```
المقرئ يدخل لوحته /reader
يشوف عداد: "X تسميع في الانتظار" (باللون الذهبي يومض)
       ↓
يضغط "ابدأ المراجعة" → /reader/recitations
يختار التسميع → يسمعه عبر AudioPlayer
       ↓
المقرئ يملأ:
  ✅ القرار: "متقن (Mastered)" أو "يحتاج جلسة (Needs Session)"
  ✅ درجة التقييم (0-100)
  ✅ ملاحظات مكتوبة تفصيلية
  ✅ الكلمات التي أخطأ فيها الطالب (Word Mistakes)
       ↓
يضغط "إرسال التقييم"
       ↓
📝 الحالة تتغير لـ: in_review → ثم mastered أو needs_session
🔔 الطالب يتلقى إشعار: "تم مراجعة تسميعك"
📧 إيميل للطالب بالنتيجة
```

---

### المرحلة 5️⃣-أ — إذا كانت النتيجة: MASTERED ✅

```
حالة التسميع: MASTERED
       ↓
🔔 إشعار للطالب: "أتقنت! تهانينا"
📧 إيميل ترحيبي بالإتقان
       ↓
الطالب يذهب لـ /student/certificate
يملأ بياناته:
  - الجامعة
  - الكلية
  - المدينة
  - الجنس
       ↓
البيانات تُحفظ في DB
       ↓
الأدمن يرى الطلب في /admin/certificates
يراجعه ويضغط "إصدار الشهادة"
       ↓
🖼️ الشهادة تُولَّد كصورة PNG (Sharp + TextToSVG):
   - اسم الطالب باللاتيني
   - كود تسلسلي: ITQ-MAK-00000001
       ↓
📤 الشهادة تُرفع على UploadThing
🔗 رابط عام: /c/[id] (قابل للمشاركة بدون تسجيل دخول)
📧 إيميل للطالب فيه الشهادة كمرفق PNG + رابط
       ↓
الطالب يقدر يحمّل الشهادة PNG أو PDF
```

---

### المرحلة 5️⃣-ب — إذا كانت النتيجة: NEEDS SESSION 📅

```
حالة التسميع: NEEDS_SESSION
       ↓
🔔 إشعار للطالب: "تحتاج جلسة تصحيحية"
📧 إيميل بالتفاصيل
       ↓
الطالب يروح /student/booking
يشوف مواعيد المقرئ المتاحة
يختار موعد مناسب
       ↓
📝 حجز يُسجَّل في DB جدول bookings: PENDING
🔔 إشعار للمقرئ: "حجز جديد في انتظار التأكيد"
       ↓
المقرئ يؤكد الحجز: CONFIRMED
🔔 إشعار للطالب: "الحجز مؤكد"
📧 إيميل بتفاصيل الجلسة
       ↓
الأدمن (أو المقرئ) يضيف رابط الاجتماع (Zoom/Meet)
📧 إيميل برابط الجلسة
⏰ كرون جوب: 24 ساعة قبل الجلسة → إيميل تذكير
       ↓
الجلسة تنعقد → الحجز يصبح: COMPLETED
حالة التسميع: SESSION_BOOKED → ثم يُعاد التقييم
```

## 👨‍🏫 ثانياً: المقرئ — ما يشوفه وما يقدر يعمله

### لوحة المقرئ `/reader`

| العنصر | التفاصيل |
|---|---|
| 🔛 مفتاح النشاط | يفعّل/يوقف استقبال تسميعات جديدة |
| 📊 KPIs | عدد المراجعات المنتظرة، جلسات اليوم، جلسات الأسبوع، حالات الإتقان |
| 📈 إحصائيات الأداء | معدل الإكمال، عدد الطلاب، متوسط التقييم |
| 🔔 طلبات مواعيد جديدة | طلاب انتهت فرصتهم ويطلبون فرصة جديدة |

### صلاحيات المقرئ:

| الصفحة | ما يشوفه/يعمله |
|---|---|
| `/reader/recitations` | قائمة التسميعات المعينة له |
| تفاصيل التسميع | يسمع الصوت + يكتب التقييم + الدرجة + الكلمات المغلوطة |
| `/reader/schedule` | جدول مواعيده المتاحة (يضيف/يحذف مواعيد) |
| `/reader/sessions` | قائمة جلساته المجدولة |
| `/reader/chat` | محادثة مع الطلاب |
| `/reader/profile` | ملفه الشخصي |

> **المقرئ لا يرى:** بيانات المنصة الكاملة، إحصائيات الأدمن، بيانات المستخدمين الآخرين

## 👑 ثالثاً: الأدمن — كل شيء

### لوحة الأدمن `/admin`

```
الأدمن يشوف في الصفحة الرئيسية:
  📊 إحصائيات فورية: (الطلاب، المقرئون، التسميعات اليوم، متوسط وقت المراجعة)
  👁️ مشاهدات الموقع (آخر 30 يوم)
  👥 الزوار الفريدون
  📈 مخطط المشاهدات والزوار عبر الزمن
  🌍 أعلى الدول بالزيارات
  📱 توزيع الأجهزة (موبايل / ديسك توب)
  🚫 الطلاب الموقوفون
  📋 أحدث التسميعات
```

### جميع صلاحيات الأدمن:

| القسم | الصلاحيات |
|---|---|
| **التسميعات** | عرض كل التسميعات + تعيين مقرئ + تغيير الحالة + إعادة التعيين + عرض الصوت |
| **المقرئون** | قبول/رفض الطلبات + إدارة المقرئين الحاليين + تعليق/تفعيل |
| **الطلاب** | عرض كل الطلاب + تعليق الحسابات + ereset كلمة المرور |
| **المستخدمون** | إدارة شاملة لكل المستخدمين |
| **الشهادات** | مراجعة الطلبات + إصدار الشهادة + إلغاء الشهادة |
| **الحجوزات** | عرض كل الحجوزات + إضافة رابط الاجتماع |
| **الإعلانات** | نشر إعلانات تصل لكل الطلاب والمقرئين |
| **قوالب الإيميل** | تعديل محتوى كل قوالب الإيميل من الواجهة |
| **الإعدادات** | SMTP + التخزين + المعلومات العامة |
| **SEO** | تعديل meta tags وبيانات الموقع |
| **النسخ الاحتياطي** | تصدير/استيراد JSON كامل لقاعدة البيانات |
| **سجل الأنشطة** | كل الأحداث في المنصة |
| **رسائل التواصل** | الرسائل من نموذج اتصل بنا |
| **التحليلات** | مشاهدات الموقع + الدول + الأجهزة |
| **الأمان** | إعدادات الأمان والصيانة |
| **وضع الصيانة** | تفعيل/إيقاف صفحة الصيانة |
| **الصفحة الرئيسية** | تعديل محتوى الصفحة الرئيسية |

## 🔐 رابعاً: المشرفون (Supervisors)

### `student_supervisor` — مشرف الطلاب
- يدخل على `/admin` بنفس واجهة الأدمن
- صلاحياته محدودة بما يخص الطلاب فقط
- **يشوف:** قائمة الطلاب، التسميعات، حالات الطلاب
- **لا يشوف:** الإعدادات الحساسة / SMTP / النسخ الاحتياطي

### `reciter_supervisor` — مشرف المقرئين
- يدخل على `/admin` بنفس الواجهة
- صلاحياته محدودة بما يخص المقرئين
- **يشوف:** طلبات المقرئين الجدد، إدارة المقرئين الحاليين
- **لا يشوف:** الإعدادات الحساسة / النسخ الاحتياطي

> **التطبيق:** المشرفون يدخلون من `/login-admin` ويروا نفس الواجهة، لكن الـ middleware يقرأ الدور ويطبق القيود.

## 📱 ملاحظات تقنية مهمة

1. **الحماية بالـ Middleware**: كل مسار محمي — الطالب ما يقدر يدخل `/reader` والمقرئ ما يقدر يدخل `/admin`
2. **الإشعارات لا تكسر التدفق**: لو فشل الإشعار، العملية الأساسية تكمل طبيعي
3. **حذف التسميع**: الطالب يقدر يحذف التسميع الخاص به بس — وبيُنظّف الملف من UploadThing اوتوماتيكياً
4. **طلبات المواعيد الجديدة**: لو انتهت مهلة الطالب لحجز موعد، يقدر يطلب فرصة جديدة من المقرئ مباشرة
